### INCORRECT Findings

1. [\[^fhe_d\]](../../../../Schemes/Group%20A%20-%20FHE%20Based%20PIR/onionpir_2021/onionpir_2021_notes.md#user-content-fn-fhe_d-d07e1b13f71a2dd99dcd0d5a440f48b5) — The footnote text states: "For N = 10^6: the formula gives d = 1 + ceil(log_4(7812.5)) = 1 + ceil(6.47) = 8." The arithmetic is wrong. log_4(7812.5) = ln(7812.5)/ln(4) = 8.964/1.386 = 6.466, so ceil(6.466) = 7, giving d = 1 + 7 = 8. The final result d = 8 is correct, but the intermediate value "ceil(6.47)" should be 7, not left ambiguous. More importantly, the footnote then says the paper "uses d = 5 for its concrete protocol" and explains this via N_1 = 128 and 4 remaining dimensions of size 4 giving 128 * 4^4 = 32768 addressable entries. This is an editorial interpretation that is misleading: the paper at Section 4.4 (p.7) says d = 1 + ceil(log_4(N/128)), and for 1 million entries with l = 5 states "a total of 386 values will be packed" into one ciphertext with n = 4096. The paper never explicitly says "d = 5" for N = 10^6. In fact, d = 1 + ceil(log_4(10^6/128)) = 1 + ceil(log_4(7812.5)) = 1 + 7 = 8 according to the formula. The footnote's claim that the paper "uses d = 5" with 128 * 4^4 = 32768 entries does not match a 10^6-entry database and is not stated in the paper. The 386 packed values are consistent with d = 8 (256 values for dim 1 + 4*5*(8-1) = 140 for higher dims = 396, or more precisely 256 + 4*5*(d-1) with the paper's stated 386). The editorial interpretation of d = 5 appears to be incorrect.

9. [\[^fhe_F\]](../../../../Schemes/Group%20A%20-%20FHE%20Based%20PIR/onionpir_2021/onionpir_2021_notes.md#user-content-fn-fhe_F-d07e1b13f71a2dd99dcd0d5a440f48b5) — The footnote says "SealPIR, in comparison, has F = 10 with q set to 60 bits and t = 12 bits." The PDF at Section 6.2, p.10 says: "SealPIR, in comparison, sets n to 2048 and q to 60 bits... They set t to 12 bits, which gives a ciphertext expansion factor of F = 10." The formula F = 2 log q / log t = 2*60/12 = 10. Correct, but the footnote's phrasing "q set to 60 bits" could be read as q = 60 (the number), when it means q is a 60-bit number. This matches the PDF phrasing exactly, so the ambiguity is inherited. Trivially minor.
